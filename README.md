# 오픈 미션: 로또 게임 - SQL 기반 리팩토링

제가 선택한 미션의 주제는 과제 설명의 미션 예시에 있는 **낯선 도구 해커톤**에 해당하며, 3주차 미션인 로또 게임을 'SQL'이라는 평소에 잘 사용하지 않았던 언어를 활용하여 구현하는 것입니다.

## 1. 미션 목표

- 3주차 로또 미션을 SQL문을 중점으로 사용하여 리팩토링
- 데이터가 데이터베이스에 저장되고 관리되는 과정을 경험하기
- 데이터의 흐름 이해 및 데이터베이스에 대한 전반적인 이해도 높이기

## 2. 미션 선정 이유

### 2.1. 풀스택 개발에 대한 관심

현재에는 프론트엔드 개발에 대한 관심이 많지만, 추후에는 **풀스택 개발에도 도전**하려는 생각을 가지고 있습니다. 아직 백엔드 개발을 해본적이 없기에, 백엔드 개발 이전에 **이해도를 높이려면** 데이터가 어떻게 흐르고 저장되는지 이해하는 것이 중요하다고 생각합니다. 이번 오픈 미션을 통해서 **SQL 구문을 직접 활용하는 실습**이 백엔드 이해에 대한 베이스가 될 것이라고 생각해서 선정했습니다.

### 2.2. 이론 학습과 실전 경험의 차이

이론으로 배우는 것과 실습을 통해 얻는 경험은 다르다고 생각합니다. 현재 전공 수업을 통해 데이터베이스 이론에 대해 배운 적은 있지만, 배운 이론을 활용해서 실제 프로그램을 만들어 본 적은 없습니다.
이번 미션을 통해 3주차 로또 미션의 각 클래스의 관계를 베이스으로 **데이터베이스 테이블을 설계**해보고, **실습을 통해 SQL 문을 활용하면서 생기는 오류를 직접 해결하는 과정**을 수행하면서 데이터베이스에 대한 이해도가 높아질 것이라고 생각해서 선정했습니다.

## 3. SQLite 데이터베이스 선택 이유

### 환경 호환성 최우선

일반적인 데이터베이스를 사용하면 다른 환경에서 실행할 때마다:

- 데이터베이스 설치 필요
- 설정(Host, User, Password 등) 입력 필요

이러한 **번거로운 과정을 제거**하기 위해 **서버가 없는 파일 기반 데이터베이스인 SQLite 3**을 선택했습니다.
